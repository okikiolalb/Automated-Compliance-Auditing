# Contributing to Automated-Compliance-Auditing

Thank you for your interest in improving architectural automation in Portugal! As we transition from the RGEU to the new 2026 Construction Code, collaboration is key to keeping this engine accurate.

## 🚀 How You Can Help
You don't need to be a coder to contribute. We value:
* **Regulatory Updates:** Identifying changes in the new "Código da Construção."
* **Bug Reports:** Finding edge cases where the Python logic fails in specific Revit room configurations.
* **Feature Requests:** Suggesting new BIM parameters to track (e.g., accessibility clearances).

## 🛠️ Development Process
1. **Fork the Repo:** Create your own copy to experiment.
2. **Branching:** Use descriptive branch names (e.g., `feat-update-art65` or `fix-indentation-error`).
3. **Pull Requests:** Submit your changes for review. Please include a brief description of why the change is necessary for Portuguese compliance.

## ⚖️ Standards
* Ensure all Python code follows **PEP 8** style guidelines.
* Any changes to regulatory values must include a reference to the specific **Decreto-Lei** or article number.
